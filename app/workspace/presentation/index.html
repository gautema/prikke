What is Runlater?

Background jobs and cron scheduling as a service

  • Cron jobs — Schedule recurring tasks with cron expressions. Daily reports, hourly syncs.
  • One-time jobs — Run once at specific time. Scheduled notifications, delayed processing.
  • Job queues — Push via API for immediate execution. Fire and forget.

Hosted in Europe · GDPR native · Data never leaves the EU

curl -X POST api.runlater.eu/v1/queue

How It Works

  • 1. Create — Define webhook URL and schedule via dashboard or API
  • 2. Execute — We make HTTP request at scheduled time. Retries included.
  • 3. Monitor — Real-time dashboard shows status, response times, history

The Tech Stack

  • Elixir + Phoenix — Fault-tolerant, concurrent. Built for systems that run forever.
  • Phoenix LiveView — Real-time UI with minimal JavaScript. Server-rendered reactivity.
  • PostgreSQL — Job queue via FOR UPDATE SKIP LOCKED. No Redis needed.
  • Tailwind CSS — Utility-first styling. Fast iteration.
  • Kamal — Docker deploys to Hetzner EU. Zero downtime.
  • Mailjet — Transactional email. French company, EU hosted.

why elixir?

BEAM VM runs forever. Lightweight processes. Fault tolerance built-in.
GenServer + Postgres = job queue. No Redis, no external dependencies.

cat CLAUDE.md

Elixir: AI-Friendly by Design

  • Phoenix ships with CLAUDE.md — Phoenix 1.8 generates agents.md out of the box. Project conventions ready for AI.
  • Inline documentation culture — @moduledoc and @doc are standard. Every function self-documents.
  • Excellent official docs — Elixir, Phoenix, Ecto, LiveView. AI references accurate info.
  • Explicit, readable code — Pattern matching is self-documenting. Pipe operator shows data flow.
  • Convention over configuration — Contexts, schemas, LiveViews. Predictable patterns AI follows.
  • Compiler catches mistakes — Warnings as errors. Dialyzer types. AI code validated immediately.

SELECT * FROM executions FOR UPDATE SKIP LOCKED

Why No Redis? No Oban?

  • Postgres can do it all — FOR UPDATE SKIP LOCKED = atomic job claiming. Advisory locks = leader election.
  • Fewer moving parts — One database to manage. No Redis cluster. Simpler = fewer failures.
  • Full control — Custom priority (Pro users first). Adaptive pool. Exactly what we need.
-- Claim next job atomically SELECT * FROM executions WHERE status = 'pending' AND scheduled_for <= now() ORDER BY priority DESC LIMIT 1 FOR UPDATE SKIP LOCKED

Lessons Learned

what worked and what didn't

echo $LESSONS_GOOD

What Worked Well

  • Exceptional on big technical tasks — Scheduler, worker pool, API auth. Complex features done fast and correct.
  • CLAUDE.md is essential — Detailed context file. Stack, architecture, conventions. AI reads and follows.
  • Tests enable velocity — 343 tests catch regressions. AI writes tests too. Pre-commit hook enforces.
  • Describe intent, not steps — Tell what you want, not how. "Add rate limiting" > step-by-step instructions.

cat ai-best-practices.md

AI Programming Best Practices

  • Give AI many tools — File access, shell commands, web search, code execution. More tools = more capable. Let it help itself.
  • Let it verify itself — AI should run its own code. See errors. Fix them. Self-correction is faster than back-and-forth.
  • Tests for every change — Tell AI: "write and run tests." Pre-commit hooks enforce. No untested code lands.
  • AI needs feedback loops — Compiler output. Test results. Runtime errors. AI improves when it sees consequences.

echo $LESSONS_BAD

What to Watch Out For

  • ⚠ Small UI tasks take longer — Pixel-perfect tweaks, alignment, spacing. More iterations than complex logic.
  • ⚠ Context gets lost — Long sessions, AI forgets earlier decisions. Re-explain or start fresh conversation.
  • ⚠ AI makes mistakes — Check edge cases. Check error handling. Check security. Review everything.
  • ⚠ Know your stack — AI accelerates, doesn't replace expertise. Guide decisions, catch bad patterns.

cat takeaways.md

Key Takeaways

  • AI is a force multiplier — Doesn't replace developers, amplifies them. Better skills = more leverage.
  • Invest in context — CLAUDE.md, docs, conventions. Upfront investment pays off in every interaction.
  • Tests are non-negotiable — AI code needs validation. Tests are safety net. Pre-commit hooks enforce.
  • Stay in the loop — Review every change. Understand the code. You own what ships to production.

The Future of Programming?

Is the developer still needed?

  • AI wrote ~90% of the code — But I directed every feature, reviewed every line, made every decision.
  • Domain knowledge matters — Knowing what to build is harder than building it.
  • Taste and judgment — When to push back, when to simplify, what's "good enough".
  • The loop still needs a human — Someone to set direction, verify correctness, own the result.
  • Shift from code to product — Less "writing code", more "delivering products". Ownership and responsibility still required.
  • Architects still needed — System design, trade-offs, integration. AI executes, humans architect.

The tools change. The craft remains.
