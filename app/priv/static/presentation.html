What is Runlater?

Product

What Runlater Does

  • Cron jobs — Schedule recurring tasks with cron expressions. Daily reports, hourly syncs.
  • One-time jobs — Run once at specific time. Scheduled notifications, delayed processing.
  • Job queues — Push via API for immediate execution. Fire and forget.

Technology

The Stack

Technology choices

Technology

The Tech Stack

  • Elixir + Phoenix — Fault-tolerant, concurrent. Built for systems that run forever.
  • Phoenix LiveView — Real-time UI with minimal JavaScript. Server-rendered reactivity.
  • PostgreSQL — Does everything. Job queue, leader election, scheduling. No Kafka, no RabbitMQ, no Redis, no SQS.
  • Tailwind CSS — Utility-first styling. Fast iteration.
  • Kamal — Docker deploys to Hetzner EU. Zero downtime.

Why Elixir?

BEAM VM runs forever. Lightweight processes. Fault tolerance built-in. GenServer + Postgres = job queue with no external dependencies.

Why no message broker?

Postgres FOR UPDATE SKIP LOCKED is a job queue. Advisory locks are leader election. No Kafka, no RabbitMQ, no Redis to manage, monitor, or pay for.

AI Development

The Workflow

  • 1. Describe — Write clear description. Context, behavior, edge cases. Better prompt = better result.
  • 2. Discuss — Back-and-forth on approach. Both contribute ideas. Architecture emerges from conversation.
  • 3. Generate — AI writes schemas, contexts, LiveViews, tests. Follows project patterns.
  • 4. Validate — Review output. Understand it. Catch mistakes. You own what ships.

Scaling the approach

Large features: Use plan mode. AI explores codebase, writes implementation plan, you approve before coding.
Small features: Just ask. Direct question, direct answer.

  • Scheduler — Ticks every 10s. Leader election via pg_try_advisory_xact_lock — only one node schedules, even across multiple servers.
  • Executions queue — Postgres table. Workers claim with FOR UPDATE SKIP LOCKED.
  • Worker pool — DynamicSupervisor. 2-20 workers based on queue depth.
  • No Redis, no Oban — Just GenServers and Postgres. Advisory locks for leader election, SKIP LOCKED for job claiming.

Lessons

What I Learned

The honest version

Lessons

What Worked Well

  • Big technical tasks shine — Scheduler, worker pool, API auth. Complex features done fast and correct.
  • CLAUDE.md is essential — Detailed context file. Stack, architecture, conventions. AI reads and follows.
  • Tests enable velocity — 371 tests catch regressions. AI writes tests too. Pre-commit hook enforces.
  • Describe intent, not steps — Tell what you want, not how. "Add rate limiting" > step-by-step instructions.

Lessons

The Uncomfortable Truth

AI requires stronger fundamentals, not weaker ones.

  • I didn't write any code — Not a single line. But I directed every feature, shaped every discussion.
  • Architecture was a collaboration — Some ideas mine, some AI's. Emerged from back-and-forth discussion.
  • 20 years of context — Two decades of knowing what good systems look like. That guides the conversation.
  • Catching mistakes requires knowledge — Session locks had connection pool bug. I spotted it because I understood the problem space.

Real collaboration. Human directs, AI executes. Both contribute ideas.

Lessons

Where AI Goes Wrong

  • ⚠ Diagnostic intuition atrophies — Temptation to prompt at first error instead of thinking through root cause. Fight this.
  • ⚠ Open-ended requests drift — "Refactor this" produces chaos. "Change X to Y" works. Be specific, stay in control.
  • ⚠ Stochastic, not sentient — AI hallucinates on small tasks. Mostly-correct but subtly wrong. Always verify.
  • ⚠ Context window is finite — Long sessions lose earlier decisions. Fresh conversations for complex changes.

The pattern that works

Targeted requests. Human direction. AI execution. Verify output. Repeat.

Lessons

Effective AI Programming

  • Give AI tools + feedback — File access, shell, tests. Let it run code, see errors, self-correct.
  • Tests are the safety net — AI writes tests too. Pre-commit hooks enforce. No untested code lands.
  • CLAUDE.md is essential — Stack, architecture, conventions, decisions. Upfront investment pays off every interaction.
  • Review everything — Understand what shipped. You own production. AI is powerful machinery, you're the pilot.

The real skill shift

From typing code to directing conversations. Experience shapes the discussion. AI executes, you steer.

Takeaway

What Does This Mean?

The transition isn't code → no-code. It's typing → directing and validating.

  • Real collaboration — Both contribute ideas. Human directs, AI executes. Architecture emerges from discussion.
  • Understanding is non-negotiable — Can't validate what you don't understand. Expertise becomes more valuable, not less.
  • The gap widens — Same AI, different results. Experienced devs get more out of it. Juniors risk learning nothing.
  • Responsibility stays human — AI doesn't own production outages. You do. Act accordingly.

The real question

Not "will AI replace developers?" but "what kind of developer thrives with AI?"
Answer: One who understands enough to validate, guide, and take responsibility.
